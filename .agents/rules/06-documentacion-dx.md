# Regla: Documentación y experiencia de desarrollo

## Nombre
Documentación ejecutable y DX verificable

## Alcance
- Aplica a: README.md, README.es.md, archivos de entorno, scripts de proyecto
- Etapa: onboarding, mantenimiento y entrega
- Responsable: quien modifique setup, comandos o variables

## Razon
Reduce fricción de entrada y evita errores por documentación desactualizada.

## Parametros claros
- referencias_archivos_inexistentes: false
- variables_entorno_documentadas: required
- guia_contribucion_tecnica: required
- scripts_desarrollo_validos: required
- decisiones_tecnicas_trazables: required

## Criterios de cumplimiento
1. Todo archivo/comando documentado debe existir y funcionar.
2. Variables de entorno deben tener ejemplo y descripción.
3. Debe existir guía corta de contribución con normas de calidad.

## Evidencia recomendada
- README coherente con el estado real.
- Archivo de ejemplo de entorno.
- Sección de convenciones y flujo de contribución.

## Proceso
- Validada.
